🔐 三、关键安全与异常设计（企业级必备）
1. 权限控制（RBAC 模型）
表格
角色	数据权限	操作权限
HR管理员	所有数据	所有操作
部门负责人	本部门及下属部门	查看/导出/收藏
企业管理层	全企业数据	仅查看
普通员工	仅个人数据	仅查看个人数据
实现：
Spring Security @PreAuthorize 注解
例如：@PreAuthorize("hasRole('DEPT_HEAD')")
查询时自动添加部门过滤条件
2. 数据安全
表格
风险	解决方案
Hive 数据泄露	SpringBoot 通过 JDBC 连接 Hive，不暴露 Hive 端口（仅开放 10000 端口）
敏感数据暴露	所有接口返回数据自动过滤敏感字段（如员工薪资）
密码安全	密码存储：BCrypt 加密（BCryptPasswordEncoder）
XSS 攻击	前端：v-html 用 v-text 替代；后端：Jsoup 过滤输入
3. 异常处理
表格
异常类型	处理方案
Hive 查询超时	设置 hive.query.timeout=300，超时返回“查询超时，请重试”
数据同步失败	自动重试 3 次，失败记录日志到 sys_error_log
权限不足	返回 403 Forbidden + 详细错误信息（“您无权查看该部门数据”）
